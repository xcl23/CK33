说明：“ASCII.txt”为95个ASCII码字符的纯文本文件；
      “unicode_file.txt”为unicode码字符的纯文本文件，它包含21000个简繁汉字；
      “gb2312_file.txt”为gb2312码字符的纯文本文件，它仅包含二级字库汉字，即6788个使用率达99%的汉字。

1.中文有UTF-8格式码，unicode码，GB2312码，其中UTF-8中文占三个字节，它是国际通用的编码方式，理论上可包含所有国家的字体码，而unicode码是中文码，仅占两个字节，它包含21000个简繁体汉字，而GB2312是二级汉字库，即国家通用的二级字库，它仅包含6788个汉字，但其使用率则高达99%，故其删除了大部分不常用的字体。

2.有关UTF-8转unicode码的方式可参见“GUI_Font_unicode.c”文件，或参见“unicode码中文字库显示方法”中说明的，它们之间的转换是有对应的公式关系，而unicode码转GB2312码的方法则参见“unicode_to_gb2312_map.c”文件，它们之间的转换是没有对应的公式的，故这里使用的是查表法，该表格是我自已写程序导出的(可参见“mainwindow.cpp”的QT程序)，即逐一读取“unicode_file.txt”文件中的每一个汉字，然后去逐一比对“gb2312_file.txt”中的每一个汉字，若其码值相同(两个字节)，则说明查找到，然后将该位置值标记为“gb2312_file.txt”中该字的检索位置值，若没查找到则表示是不常用的汉字，该位置就用0xFFFF值代替，这样所有unicode_file.txt文件中的字全部查找一遍后就可生成一个位置映射表，将该表建一个数组，数组个数即为21000大小，每一个元素值即表示“gb2312_file.txt”中该汉字对应的索引位置值，这样的映射表使用时只需将unicode码的index值代入数据中去，取其值即表示对应“gb2312_file.txt”文件中的汉字位置值，这个映射表大小为42K左右，但二级汉字库仅为全汉字库的1/3，故其空间还是非常小的，且该映射表是可以对不同字体大小的汉字都适用，若代码中包含多种字体，那么这种方法是非常省空间的，故不
考虑繁体字的情况下可使用二级汉字库。

3.汉字库取模，汉字库取模要用软件III3.9来做，具体方法可参见“unicode码中文字库显示方法”来做，这里仅说明一点是若要生成GB2312码，则导入文件就用“gb2312_file.txt”，生成的c文件即为汉字字模码，若要生成unicode字模，则导入文件用“unicode_file.txt”，同时还应生成ascii字符字模，可导入文件“ASCII.txt”，因为ascii显示大小与汉字不同，故这里将ascii字模单独取模，而不是放在一起，那样显示的ascii字体会很宽，不好看，当导出汉字字模码后，可将ascii码字模文件全部复制到汉字字模码的开头位置处，再加上对应的结构体名，就算做成了一份新的字模文件了，它包含英文及中文字体。
